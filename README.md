# 学習商品専用 LP フレームワーク v3

この v3 は、舞台LPの素材や世界観を流用するものではありません。参考にするのは、PC版の「左に大きなビジュアル、右に本文、右側の上下ボタンで1画面ずつ切り替える」という型だけです。

ACTS / ABDT など、学習商品LPを短期間で量産するための HTML フォーマットとして使う想定です。

## この型で固定すること

- PCでは左側に大きなビジュアル、右側に本文を置く
- 右側は通常スクロールではなく、上下ボタンまたはタップ操作で1シーンずつ切り替える
- 本文が切り替わると、左側のビジュアルも同期して切り替わる
- スマホでは上にビジュアル、下に本文を置き、同じくシーン単位で切り替える
- 画像やコピーは学習商品ごとに差し替える

## ファイル構成

- `learning-product-lp-framework.html`
  - そのまま開ける単体HTMLのLPフレームワーク
- `lp-data.schema.json`
  - 51本分のLP情報を整理するためのデータ型
- `assets/README.md`
  - 画像差し替え時の置き場所メモ

## PC版の考え方

PCでは、画面全体を左右に分けます。

- 左側: `visual-rail`
  - 画面高さいっぱいの大きなビジュアル領域
  - 各シーンの `visual` に合わせて画像を切り替える
  - 参考LPのように、本文の内容に合わせて視覚印象も変える
- 右側: `content-rail`
  - 黒系の読み物エリア
  - Hero、共感、学習価値、内容、申込導線、FAQ などを1シーンずつ表示する
  - 右端の上下ボタンで前後のシーンへ移動する

重要なのは、右側を長くスクロールして読むLPではなく、「1画面ずつ見せるLP」にすることです。

## スマホ版の考え方

スマホでは、上にビジュアル、下に本文を置きます。

- 上: 現在シーンに対応するビジュアル
- 下: 現在シーンの本文
- 操作: 上下ボタン、キーボード、スワイプ
- 文章が長い場合だけ、シーン内で最小限の縦スクロールを許容

## 51本量産時に差し替える情報

- 商品名
- ACTS / ABDT などのエンジン
- SMP
- Heroコピー
- 共感コピー
- 学習価値
- 教材内容
- 対象年齢
- 申込ボタン文言 / URL
- FAQ
- 価格、キャンペーン、注意書き
- 各シーンの画像

価格や会員数などの機微情報は、8tdbではなく SharePoint 等の正式データを確認して差し込んでください。

## 使い方

`learning-product-lp-framework.html` 内の `LP_DATA` を商品ごとに差し替えます。

各 `sections` に `visual` を指定すると、そのシーンを表示した時に左側画像も自動で切り替わります。画像を切り替えたくないシーンでは、同じ画像パスを指定してください。
